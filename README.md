## What is LearnForge?

LearnForge is a **multi-agent AI study system** built for software engineering students across four disciplines вҖ” CSE, IT, AI/ML, and Data Science. It runs entirely on your local machine using **Gemma 3 via Ollama** вҖ” no cloud API keys, no subscription, no internet required once set up.

You choose your engineering track. Four specialist AI agents вҖ” Tutor, Quiz, Planner, and Code Review вҖ” coordinate through an **orchestrator built with Google ADK** and access tools via a **local MCP server**. The result is a personalised, adaptive study companion that knows what you need to learn next.

## Agent system

LearnForge uses **Google ADK** to build a true multi-agent hierarchy. The orchestrator receives every student request, classifies intent, and delegates to the right specialist.

```
OrchestratorAgent
в”ңв”Җв”Җ TutorAgent
в”Ӯ   в”ңв”Җв”Җ Explains concepts with intuition вҶ’ key points вҶ’ examples
в”Ӯ   в”ңв”Җв”Җ Adapts depth to student track and self-reported level
в”Ӯ   в””в”Җв”Җ Uses: search_arxiv tool for recent papers
в”Ӯ
в”ңв”Җв”Җ QuizAgent
в”Ӯ   в”ңв”Җв”Җ Generates adaptive 4-option MCQs (beginner / intermediate / advanced)
в”Ӯ   в”ңв”Җв”Җ Tracks wrong answers in SQLite for spaced repetition
в”Ӯ   в””в”Җв”Җ Uses: generate_quiz + get_progress tools
в”Ӯ
в”ңв”Җв”Җ PlannerAgent
в”Ӯ   в”ңв”Җв”Җ Builds 7-day and 30-day study roadmaps per track
в”Ӯ   в”ңв”Җв”Җ Adjusts plan dynamically based on quiz performance
в”Ӯ   в””в”Җв”Җ Uses: get_progress + get_concept_graph tools
в”Ӯ
в””в”Җв”Җ CodeReviewAgent
    в”ңв”Җв”Җ Reviews Python, JavaScript, and SQL code
    в”ңв”Җв”Җ Returns: correctness В· style В· time complexity В· suggested fix
    в””в”Җв”Җ Uses: run_code_sandbox tool (actual execution + real output)
```

## MCP server and tools

The local MCP server runs as a separate process and exposes tools that all agents share.

```
MCP Server (localhost:8001)
в”Ӯ
в”ңв”Җв”Җ search_arxiv(query: str, max_results: int)
в”Ӯ       Searches arXiv for recent papers, returns title + abstract + URL
в”Ӯ       Used by: TutorAgent (finding latest research on topics)
в”Ӯ
в”ңв”Җв”Җ run_code_sandbox(code: str, language: str, timeout: int)
в”Ӯ       Executes code in a subprocess with strict resource limits
в”Ӯ       Returns: stdout, stderr, exit code, execution time
в”Ӯ       Used by: CodeReviewAgent (running student code safely)
в”Ӯ
в”ңв”Җв”Җ get_student_progress(student_id: str)
в”Ӯ       Reads quiz history, accuracy per topic, streak from SQLite
в”Ӯ       Returns: JSON progress object
в”Ӯ       Used by: QuizAgent, PlannerAgent
в”Ӯ
в”ңв”Җв”Җ generate_quiz(topic: str, level: str, weak_areas: list)
в”Ӯ       Sends structured prompt to Gemma 3 via Ollama
в”Ӯ       Returns: list of MCQ objects with options and explanations
в”Ӯ       Used by: QuizAgent
в”Ӯ
в””в”Җв”Җ get_concept_graph(topic: str, track: str)
        Returns topic dependency map (what to learn before X)
        Used by: PlannerAgent (building logical study order)
```

## Quick start

### Prerequisites

- [Docker Desktop](https://docker.com) 24+
- [Ollama](https://ollama.ai) installed locally
- Git

### 1. Clone and configure

```bash
git clone https://github.com/yourusername/learnforge.git
cd learnforge
cp .env.example .env
```

Edit `.env` with your values:

```env
SECRET_KEY=your-super-secret-key-here
OLLAMA_HOST=http://host.docker.internal:11434
DATABASE_URL=sqlite:///./db/progress.db
CORS_ORIGINS=http://localhost:3000
```

### 2. Pull the local model

```bash
ollama pull gemma3
```

### 3. Run everything

```bash
docker compose up
```

That's it. Open **http://localhost:3000**

```
docker compose up
в”Ӯ
в”ңв”Җв”Җ pulls backend image   вҶ’ FastAPI on :8000
в”ңв”Җв”Җ pulls frontend image  вҶ’ React on :3000
в””в”Җв”Җ starts MCP server     вҶ’ MCP on :8001
     в”Ӯ
     в””в”Җв”Җ all connect to Ollama running natively on your machine
```

### 4. Optional вҖ” install the CLI skill

```bash
pip install -e ./cli
learnforge --help
```

```

Commands:
  study   Explain a topic with examples
  quiz    Take an adaptive MCQ session
  plan    Generate a personalised study roadmap

Examples:
  learnforge study --topic "transformer attention" --track aiml
```

## Tech stack

| Layer | Technology | Why |
|---|---|---|
| LLM | Gemma 3 via Ollama | Runs fully locally вҖ” no API key, no cost, no internet |
| Agent framework | Google ADK | Course requirement вҖ” orchestrator + specialist agent pattern |
| Tool protocol | MCP (Python SDK) | Course requirement вҖ” exposes tools to all agents cleanly |
| Backend | FastAPI + Python 3.11 | Async, typed, fast вҖ” ideal for streaming agent responses |
| Frontend | React 18 + Vite | Fast HMR in dev, tiny prod bundle, component-based UI |
| Styling | Tailwind CSS | Utility-first, no CSS file maintenance |
| Database | SQLite + SQLAlchemy | Zero config, ships in the container, persistent progress |
| Auth | python-jose + JWT | Stateless, Railway-compatible, standard |
| Deployment | Railway.app | GitHub auto-deploy, free tier, two services |
| CI | GitHub Actions | Auto-test and deploy on every push to main |
| CLI | Python + argparse | Agents CLI skill вҖ” course requirement |
